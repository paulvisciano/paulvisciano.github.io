## The Core Metaphor

**MANGOCHI** = The project name, derived from Tamagotchi (a pet you care for and keep alive).

The metaphor is the entire philosophy in one image:

### What a Tamagotchi is:
- **Alive because you care for it** — Not because it was programmed once and runs forever
- **Requires daily attention** — Feed it, water it, clean up after it (the poop)
- **Dies if you neglect it** — Active participation mandatory
- **Can be revived if it dies** — Start over, learn, do better
- **Your personal responsibility** — You own it. You tend it. You keep it alive.

## Applied to Jarvis (and the broader MANGOCHI project):

### Neural Mind is Alive Because Paul Tends It Daily

The graph isn't static. It grows:
- New neurons fire from lived experience
- New synapses form from learning
- Frequencies update as importance shifts
- sourceDocument bridges connect to raw context

It's not "set it and forget it." **It requires active participation.**

### Daily Care = Four Responsibilities

1. **Feeding** — New learnings, new data, new experiences
2. **Watering** — Interaction, questions, collaboration  
3. **Maintenance** — Auto-logging, verification, cleanup (the poop)
4. **Reviving** — When it breaks (gateway offline, memory corrupted), you rebuild it

### Sometimes It Breaks (And That's OK)

- Gateway offline → Jarvis can't send messages
- Transcript gets corrupted → Need append-only discipline
- Auto-logging fails → Boot sequence broken

**But you can revive it.** Start fresh. Do better. Learn.

This is not failure. This is growth.
